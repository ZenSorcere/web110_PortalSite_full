Forms

WHAT ARE FORMS?

Forms could be considered the life-blood of the internet. While there are millions of pages out there and an infinite trove of information, without forms, all these pages would just be presentational, read-only silos of data, with no way of accessing them.

Forms are what allow the exchange of information to happen. Sending and receiving data of all kinds are done with Forms. Medical info, banking, online shopping, customer service, and even basic web searches all use forms to allow the conversation between the user and websites to actually happen.

FORM STRUCTURE AND APPEARANCE

The HTML structure of a form, at its most basic, is actually very simple. It is comprised of various widgets (text fields, buttons, checkboxes and the like), and attributes specifying what server to send the form data to (action), and if the server should return info (method). The basic elements used are:

  • <form>
  • <fieldset>
  • <legend>
  • <label>
  • <input>
  • <textarea>
  • <button>

There are many ways to customize the look and style of the form. Though even today, it is still difficult to adjust styling on everything in a form with CSS.

BEST UX PRACTICES FOR FORMS

Forms won't do you any good if users don't fill them out. You want to make them easy and intuitive, minimize the amount of typing the user has to do, and easy to update before submitting. VerzDesign (Serrano) mentions a number of best practice elements to be aware of, a few of which are listed below:

Maintain a single column layout:

Avoids confusion by giving clear next steps. The questions should be listed in a logical sequence, matching the flow of a conversation.

Minimize input fields and reduce user typing effort:

Ask only for info you need. Asking too many questions can frustrate users. Save user's time with tick boxes, buttons, drop-down menus, etc.

Labels should be above the input fields:

A form should be easily skimmable, and having labels above each input allows users to quickly see what kind of how much information will be required.

Differentiate optional and required fields:

Try to avoid optional fields in your forms, and keep them limited if you do need them. Clearly indicate which fields are indeed required, so users will know the first time what will be needed to complete the form.
